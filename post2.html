Everyday Unix Commands


Disclaimer: I am not an expert or even an advanced Linux user. I know only the basics, so that will be the focus of this post.

Linux has come a long way in the last 10 or 15 years. Gone are the days of spending hours reading posts on obscure message boards and searching for firmware - on the family computer, of course, since usually some of the drivers you were missing were for your laptop's wireless card. Still, the vast majority of personal computers today are running Windows or Mac, so many people performing data science or other programming-heavy tasks on their own computers don't have much experience with a Unix command line. By the way, Linux and Unix are not exactly synonymous: Linux is a Unix-like operating system, as is Mac, while Windows isn't. Therefore, Mac and Linux users can simply open up a terminal and use any of these commands, but the setup for Windows users is a little more involved.

Directory navigation

pwd: print working directory. Displays the absolute path to the directory (folder) in which you're currently working. This is really helpful when you get lost because you were in a hurry and gave crappy names to everything.



ls: list. Prints a list of directories and files in the current working directory. By default it does not include hidden directories and files (these start with . on Linux systems), but by specifying -a you can show hidden items. -o gives more output, including the date and time that each item was last modified. Finally, if you specify either a relative or absolute path to a directory, ls will list the contents of that directory rather than the current working directory.



cd: change directory. With no arguments it will take you to your home directory; otherwise, give it a relative path to the directory you want to go to. cd ../ will move you up one level, and you can go up several levels at a time this way. If you try to give it a path to a file (like ./index.html, where ./ means "in the current directory") you'll get an error. If you type ls and hit tab twice, as with any Unix command, you'll get a list of options for arguments you can specify.



Manipulating directories and files

touch: Create a file or files in the current working directory. To create several files, list them separated by spaces.



rm: remove. Remove files. By default, you won't get a prompt asking if you're sure you want to remove the file before going through with it, so make sure you have the right one. You can use -i to give you a prompt, but since it will just repeat the names of the files you just typed, it may not be very useful. You can use rm on directories too (see the entry for rmdir below).



mkdir: make directory. Create a new empty directory or directories; analogous to touch. You can create a new directory and cd into it in one line using a semicolon (you can always execute multiple commands at once with this method).



rmdir: remove directory. If there's anything in it, you will need to use rm -rf instead. You can remove several empty and non-empty directories together this way. This can be dangerous for obvious reasons, so make sure there's nothing in any of them that you need beforehand.



mv: move. Use mv to move a file or a directory to a new location. It will work even if the directory you're trying to move isn't empty, which will break any code that uses the old path to that directory. A non-obvious use of this command is renaming files: mv oldfile.txt newfile.txt will create newfile.txt, move the contents of oldfile.txt into it and remove oldfile.txt, producing the same effect as right click -> rename.



cp: copy. Similar to mv except that the file you're copying is now in its original location as well as the location specified. Can be used to copy directories too, using -r.



Other handy commands, notes

man: Prints the manual page for the command specified. Press q to exit. Really great if you want a quick list of all of the options you can give to a command, but for practical examples Google is usually more useful. Running man man will give you the man page for the man command. See what happens if you try running man man man!

clear: Clears the output of your terminal. Usually better than opening a new window since you can scroll back up to see previously run commands.

Note: * serves as a wildcard, so, for example, running rm -rf ./* will clear out everything in the current directory. This is an extremely bad idea because if you forget the period you will delete your entire filesystem and render your computer unusable (at least on Linux).

That's all I've got for now. This is not even close to an exhaustive list of useful Unix commands, but it is most of the ones I use every day. I hope it was helpful. Thanks for reading.
