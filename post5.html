This is a simple example, but it highlights one of the crucial differences between so-called scripting languages like Python and "old school" languages like C: Python is an interpreted language, which means you can write a line of code and execute it immediately with just a press of the return key, whereas C is compiled, meaning you have to save your code in a .c file, compile it and only then can execute it. Compiled languages tend to be much faster, which can be important when dealing with huge quantities of data or doing strenuous physics calculations, but interpeted languages tend to be a lot more convenient for tasks where you're experimenting, running code and checking the results and trying something different and fixing mistakes along the way - like exploratory data analysis.

Another major difference between C and Python is the huge number of Python libraries and other tools to facilitate any task you can think of. The big advantage that C and other compiled languages (C++, Java, FORTRAN) have over interpreted languages (Python, Ruby, JavaScript) is speed. A lot of Python libraries are in fact wrappers that allow Python to call functions implemented using C. Some of the performance-critical parts of pandas are written in C, for example.
